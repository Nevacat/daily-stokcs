# 보안 리뷰 스킬

변경된 코드(git diff 기준)를 대상으로 보안 점검을 수행한다.

## 절차
1. `git diff` (또는 지정된 파일)로 변경 범위를 파악한다.
2. [checklist.md](checklist.md)의 항목을 순서대로 점검한다.
3. 발견 사항을 심각도(Critical/High/Medium/Low)별로 정리해 보고한다.
4. 각 발견 사항에는 파일:라인, 문제 설명, 수정 제안을 포함한다.

## 원칙
- 확실하지 않은 항목은 "의심"으로 표시하고 근거를 남긴다.
- 수정은 사용자가 요청할 때만 수행한다 (리뷰와 수정을 분리).
