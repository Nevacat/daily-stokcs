# 뉴스 기반 종목 추천 앱 기획서

- **작성일**: 2026-07-22
- **버전**: v0.1 (초안)
- **프로젝트 코드명**: (가칭) StockRadar

---

## 1. 프로젝트 개요

### 1.1 한 줄 정의
뉴스를 일정 주기로 수집·분석하여 **분야(섹터)별 유망 종목을 추천 이유와 함께** 보여주는 앱.

### 1.2 문제 정의
- 개인 투자자는 매일 쏟아지는 뉴스 속에서 "어떤 뉴스가 어떤 종목에 호재/악재인지" 판단하기 어렵다.
- 뉴스 → 종목 연결에는 시간과 배경지식이 필요하고, 놓치면 타이밍을 잃는다.
- 기존 종목 추천 서비스는 "왜 추천하는지" 근거가 불투명해 신뢰하기 어렵다.

### 1.3 핵심 가치 제안
1. **자동화**: 일정 주기로 뉴스를 자동 수집·분석 (수동 새로고침도 지원)
2. **투명성**: 추천마다 근거 뉴스와 추천 이유를 함께 제시
3. **분야별 정리**: 반도체, 2차전지, 바이오 등 섹터 단위로 정리된 추천

### 1.4 타깃 사용자
- **주 타깃**: 뉴스 기반으로 투자 아이디어를 얻고 싶은 20~40대 개인 투자자
- **부 타깃**: 특정 섹터(예: AI, 방산)를 집중 추적하는 테마 투자자

## 2. 핵심 기능 (요청 기반)

### 2.1 뉴스 수집 (F-01)

| 항목 | 내용 |
|---|---|
| 자동 수집 | 설정된 주기(기본 1시간, 옵션: 30분/1시간/3시간/6시간/1일)마다 뉴스 수집 |
| 수동 수집 | "지금 수집하기" 버튼으로 즉시 수집 + 분석 실행 |
| 수집 대상 | 경제/증권 카테고리 뉴스 (뉴스 API 또는 RSS 피드) |
| 중복 처리 | 제목/URL 유사도 기반 중복 기사 병합 |
| 상태 표시 | 마지막 수집 시각, 다음 수집 예정 시각, 수집 진행 상태(수집 중/분석 중/완료) 노출 |

**동작 흐름**
```
[스케줄러 or 수동 버튼]
  → 뉴스 수집 (API/RSS)
  → 중복 제거·정제
  → 분석 (섹터 분류 + 종목 매핑 + 호재/악재 판단)
  → 추천 결과 생성 및 저장
  → UI 갱신 (+ 알림)
```

### 2.2 분야별 종목 추천 (F-02)

- 추천 결과를 **섹터 탭/그룹**으로 묶어서 표시
- 기본 섹터(초기 8개 제안): 반도체/AI · 2차전지 · 바이오/헬스케어 · 자동차 · 금융 · 엔터/콘텐츠 · 방산/조선 · 에너지/화학
- 섹터별 추천 종목 수: 최대 3~5개 (과다 추천 방지)
- 각 종목 카드에 표시할 정보:
  - 종목명 / 티커
  - 추천 등급 또는 점수 (예: 강한 긍정 ~ 약한 긍정, 0~100점)
  - 추천 시각
  - 관련 뉴스 개수

### 2.3 추천 이유 제시 (F-03) — 차별화 핵심

- 종목 카드 탭 → 상세 화면에서:
  - **추천 요약**: "○○ 수주 소식 + 정부 정책 수혜 기대" 식의 1~2문장 요약
  - **근거 뉴스 목록**: 이 추천의 근거가 된 뉴스 링크 (각 뉴스에 긍정/부정 태그)
  - **판단 로직 공개**: "긍정 뉴스 4건, 부정 1건 → 종합 점수 78점" 처럼 산출 과정 노출
- 신뢰를 위해 **면책 문구** 상시 노출: "본 정보는 투자 참고용이며, 투자 판단의 책임은 이용자에게 있습니다."

### 2.4 뉴스 리스트 (F-04)

- 수집된 전체 뉴스를 최신순 리스트로 제공
- 각 뉴스 항목: 제목, 언론사, 발행 시각, 관련 섹터 태그, 관련 종목 태그, 긍정/부정/중립 배지
- 필터: 섹터별 / 종목별 / 감성(호재·악재)별
- 뉴스 탭 → 원문 링크로 이동 (외부 브라우저 또는 인앱 웹뷰)

## 3. 추가 제안 기능 (상품성 강화)

우선순위 순으로 제안합니다.

### 3.1 관심 종목 / 관심 섹터 (P1)
- 사용자가 종목·섹터를 즐겨찾기 → 홈 상단에 우선 노출
- **가치**: 재방문 유도의 기본 장치. 개인화의 출발점.

### 3.2 푸시 알림 (P1)
- 관심 종목/섹터에 새 추천 또는 중요 뉴스 발생 시 알림
- 알림 옵션: 전체 추천 갱신 알림 / 관심 항목만 / 끄기
- **가치**: "자동 수집" 기능의 효용을 완성함. 수집해도 사용자가 모르면 의미가 없음.

### 3.3 추천 히스토리 & 적중률 (P1)
- 과거 추천 내역을 날짜별로 보관
- 추천 시점 대비 현재 주가 등락률을 함께 표시 → "이 앱의 추천이 실제로 맞았는지" 검증 가능
- 섹터별/전체 적중률 통계 대시보드
- **가치**: 추천 서비스의 신뢰도를 데이터로 증명하는 강력한 차별화 요소. 유료화의 근거가 됨.

### 3.4 데일리 브리핑 (P2)
- 매일 아침 1회 "오늘의 시장 요약 + 주목 섹터 + 추천 Top 3" 카드 생성
- **가치**: 매일 아침 앱을 여는 습관 형성 (리텐션 핵심).

### 3.5 뉴스 감성 트렌드 차트 (P2)
- 종목/섹터별로 최근 7일간 긍정·부정 뉴스 비율 추이를 미니 차트로 표시
- **가치**: "지금 분위기가 좋아지는 중인지 나빠지는 중인지" 흐름을 한눈에 파악.

### 3.6 추천 공유 (P2)
- 추천 카드(종목 + 이유 + 근거 뉴스)를 이미지/링크로 공유
- **가치**: 바이럴 유입 채널. 투자 커뮤니티 공유 문화와 잘 맞음.

### 3.7 프리미엄 구독 (P3, 수익 모델)
- 무료: 수집 주기 6시간 고정, 섹터당 추천 1개, 히스토리 7일
- 프리미엄: 수집 주기 30분, 전체 추천, 히스토리 무제한, 실시간 알림
- **가치**: 운영비(API·분석 비용) 회수 및 수익화 경로.

```
[하단 탭 4개]
├── 홈 (추천)
│   ├── 마지막 수집 시각 + [지금 수집하기] 버튼
│   ├── 관심 종목/섹터 추천 (있을 경우 상단 고정)
│   ├── 섹터 탭 (반도체 | 2차전지 | 바이오 | ...)
│   └── 종목 추천 카드 리스트
│         └── [종목 상세] 추천 이유 + 근거 뉴스 + 감성 트렌드
├── 뉴스
│   ├── 필터 (섹터/종목/감성)
│   └── 뉴스 리스트 → 원문 보기
├── 히스토리
│   ├── 날짜별 과거 추천
│   └── 적중률 통계
└── 설정
    ├── 자동 수집 주기 설정 (30분~1일 / 끄기)
    ├── 알림 설정
    ├── 관심 종목/섹터 관리
    └── 면책 조항·약관
```

## 5. 추천 로직 설계 (개요)

1. **수집**: 뉴스 API/RSS에서 경제·증권 뉴스 수집
2. **전처리**: 중복 제거, 본문 요약 추출
3. **분석** (LLM 활용 권장):
   - 기사 → 관련 섹터 분류
   - 기사 → 관련 종목 추출 (종목명 사전 매칭 + LLM 보정)
   - 기사 → 감성 판단 (긍정/부정/중립 + 강도)
4. **점수화**: 종목별로 `Σ(뉴스 감성 점수 × 최신성 가중치 × 언론사 신뢰 가중치)` → 0~100점
5. **추천 생성**: 섹터별 상위 N개 종목 선정 + LLM으로 추천 이유 문장 생성
6. **저장**: 추천 스냅샷 저장 (히스토리·적중률 계산용)

> 기술 스택, 뉴스 소스(네이버 뉴스 API / RSS 등), LLM 선택은 개발 설계 단계에서 확정.

## 6. 정책 및 리스크

| 항목 | 내용 |
|---|---|
| 법적 리스크 | "투자 자문"이 아닌 "정보 제공" 서비스로 포지셔닝. 면책 조항 상시 노출. 매수/매도 지시 표현("사세요") 금지, "긍정 뉴스가 많습니다" 식 사실 전달 톤 유지 |
| 뉴스 저작권 | 기사 본문 전재 금지. 제목 + 요약 + 원문 링크만 제공 |
| 분석 오류 | LLM 오분류 가능성 → 근거 뉴스를 항상 함께 노출해 사용자가 직접 검증 가능하게 |
| API 비용 | 수집 주기·분석량과 비용이 비례 → 무료 티어 주기 제한으로 통제 |
